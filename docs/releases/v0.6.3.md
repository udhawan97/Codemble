# Codemble v0.6.3 — guidance and navigation that keep their place

v0.6.3 closes the four follow-up findings from the 2026-07-21 user-flow audit.
It is a focused interaction release: parser evidence, graph ordering, checks,
progress, provider behavior, and the Formal Edo visual system are unchanged.

## Highlights

- Easy guidance is level-aware. It opens the suggested system, offers **View
  structures** when the Map cannot draw that system's internals, then becomes a
  prose instruction in the Galaxy instead of leaving an enabled no-op button.
- Compact Maps open at readable 100% around Home or the selected target. **Fit**
  is still available for a whole-diagram overview, and zoom/pan now survive map
  refreshes and layer remounts.
- Compact **Switch project** reveals its confirmation on the first click and
  cannot leak an open Menu across project changes or responsive breakpoints.
- Keyboard focus moves into Home calibration, Modules, Find, and the Star chart,
  then returns to the invoking action or the visible compact Menu. Finishing the
  first-run coach hands focus to the project map.

## Verification

- Python tests and Ruff
- Web projection, session, viewport, focus, and responsive-flow contracts plus
  the production SPA build
- Astro validation and production documentation build
- Real bundled-app runs at 320, 375, 768, 1280, and 1440 px, including compact
  project switching, audience → Home → coach focus, global surface focus,
  guidance transitions, Map target sizing, zoom/pan remount persistence,
  responsive Menu lifecycle, reduced motion, overflow, and console checks
- Graphify incremental refresh and a scoped post-refresh flow query
- Wheel/sdist build, isolated install, checksums, CI, Pages, GitHub release
  assets, and public PyPI `uvx` smoke

## Known limits

- **Fit** is an overview and may make labels and targets too small to use; press
  the percentage button to return to readable 100%.
- The 3D Galaxy still requires WebGL; the parser-backed Map remains the fallback.
- The Phase 1 milestone still requires human tester evidence in issue #13.
