# v0.6.4 — the learning loop, audited end to end

Twenty findings from a fresh user-flow audit of the served build, all fixed and
re-verified on the real surface. The audit drove three real projects through the
real picker (Codemble at 109 systems, Golavo at 411, FolioOrb at 4,899 nodes) at
1280, 375 and 320 CSS pixels, with a keyboard-only pass, provider keys stripped
so the no-key state was exercised, and an isolated `CODEMBLE_DATA_DIR`.

## The one that matters most

**A missed check handed back its own answer.** Submitting a wrong option printed
`Parser answer: …` and `Evidence: …`, and the same question then accepted that
answer — so a region could be lit by replaying what the screen had just shown.
Illumination is the product's core claim, so this is the same severity class as
the 2026-07-19 "every check must offer a wrong option" fix.

A miss now returns no answer, no answer labels and **no evidence**: an importer
check cites exactly the files that are its answer, so returning the citation was
the same reveal by another name. All three come back once the learner answers
correctly. Pinned by `test_a_missed_check_never_hands_back_its_own_answer`.

## Easy mode can now read before it is asked to prove

Easy mode lands on the Map, which offered checks but had no way to show a
module's source. Region focus now carries **Read the source** beside **Prove
understanding**, Easy guidance says read-before-prove instead of routing the
learner to another layer, and Escape steps back a level on the Map as it always
has in the Galaxy. No new truth: the study panel is layer-neutral, so the Map
only had to select the module node the parser already produced.

## Keyboard

- Opening the checks panel hands focus into it (it sits after the whole map in
  DOM order — reaching question one previously cost ~114 tab stops), and
  submitting keeps focus on the result instead of dropping it to `<body>`.
- At study level, Enter now opens the structure the arrow keys announced. It was
  a no-op, contradicting the view's own instructions.
- Escape closes the star chart and returns focus to the control that opened it.

## First run

- The audience question is asked **once per learner**, not once per project; a
  new project seeds from the last answer and the header toggle still overrides
  any single project.
- Home calibration is a native modal sized to the window (it was capped to a
  share of a short stage, showing one candidate of eleven), states how many
  candidates exist, groups them by their real top-level scope so test fixtures
  no longer sit unmarked beside your entrypoint, and keeps "Explore without
  Home" on screen instead of thousands of pixels below the fold.
- Guidance no longer appears during the audience, Home and coach steps, where it
  used to recommend a target "because no import route reaches it from Home"
  while the breadcrumb still said Home unselected.

## Reading the sky

- The guidance chip is docked into its own strip above the footer. As an overlay
  it covered the Map's teaching caption, galaxy name plates, and its own button.
- Galaxy plates use the path-tail rule the Map's boxes already use, so two
  different `__init__.py` modules are no longer identically labelled.
- Find opens on Home and the busiest modules instead of a screen of identical
  `__init__.py` rows.
- The star chart keeps its header slot at every level, and every exit names the
  layer it returns to ("Back to map" / "Back to galaxy").
- Easy guidance prefers a substantive module among equal-hop candidates, instead
  of the alphabetically-first four-line package init.

## Smaller, still real

- The study panel no longer shows "Used by 0" above a summary saying five other
  parts use the file: the call-edge stat is "Called by", leaving "use" to
  imports.
- Compact Map controls no longer share touch targets — measured at 320 and 375,
  zero overlapping interactive rectangles (the Key button covered 40% of the
  "What runs first" tab).
- The exception-handling lens note is true of `raise` as well as `try`/`except`,
  since the parser files all three under one concept.
- Counts of one read as one; the picker opens beside your last project.

## Known limits

- The staged parse-progress screen and its cancel path were **not** exercised:
  every project in the audit parsed faster than one automation round-trip. The
  scale-cap prompt and the crashed-worker error state likewise remain covered by
  server tests only.
- Reduced motion and the ~1.2 s light-up animation were verified as a state
  change, not visually.
- Chromium only; explanations were exercised in the no-key state only.
- Issue #13 still requires human tester evidence: this release does not advance
  the Phase 1 milestone.
