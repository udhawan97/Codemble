# v0.7.0 — constellations you can tell apart

Released 2026-07-22 · tag `v0.7.0` · frontend-only: parser, graph, checks,
progress, and HTTP contracts are byte-unchanged from v0.6.4.

This release implements all fourteen findings of a fresh evidence-based
user-flow audit of the served v0.6.4 build (run as a first-time Easy learner on
this repository at 1280/375/320), plus the approved design direction that came
with it: **hue now answers "which part of the project is this?"**.

## Highlights

- **Community colour families (D1).** Each parser-proven import community wears
  one of eight traditional Japanese hues — seiji, fuji, koke, asagi, toki,
  umenezumi, wakatake, kikyō — deterministically from its community id. Stars
  keep centrality as lightness within the family; planets inherit their
  system's family; Architecture boxes tint toward it. All eight tokens are
  lightness-tuned to the unlit ceiling's luminance (measured 8.1:1 on
  `--cm-ground`, 7.0:1 on `--cm-ground-2` as legend swatches) while a lit
  amber star stays ~54% brighter; the amber band is excluded from the wheel.
  Amber keeps its monopoly on understanding, ruri on interaction, and
  uncertainty keeps its own channel.
- **Visible connections.** A dedicated route ink (4.0:1) on both layers
  replaces the border hairline that made every edge near-invisible (the galaxy
  additionally multiplied it to 32% opacity). Possible relationships remain
  dashed on the map and deliberately the more visible claim.
- **The unreached shelf (D3).** Architecture-map modules with no import route
  from Home fold behind their exact count with a **Show them** control. On this
  repository that removes 80 fixture/script boxes from the default geometry —
  from a 7% unreadable Fit to a readable connected core — while every module
  stays counted and reachable.
- **Mechanical fixes caught by the audit:** the nebula dawn no longer squashes
  the lit system's name plate (scale restored as a vector, not a scalar); the
  map's language stripe renders (SVG `fill` attribute cannot resolve `var()`;
  it now paints via style); Escape steps back on the Map from any focus
  position; stale viewports re-centre on the focus after resizes; Fit
  fits width on tall drawings; the Key no longer covers the zoom controls; the
  region panel's actions no longer clip.
- **Easy speaks Easy.** "candidate 1" (not "parser rank 0"), "Quiz · answers
  come from your code, not AI", a lit celebration that explains invalidation
  in plain words, "What it is / Length / Evidence" study labels, fixture
  errors attributed to their directory, and guidance that charges test-scoped
  paths a bounded +1.5-hop penalty so a learner's own code comes first.

## Evidence

- 241 pytest tests, `ruff check` clean, all 12 frontend contract checks green
  (including the new `check_galaxy_effects.mjs`), production SPA rebuilt and
  committed.
- Every audit finding re-verified on a fresh first run of the rebuilt bundle:
  before/after screenshots recorded in the audit report generated this
  session.
- App console: 0 errors, 0 warnings across the full flow.

## Known limits

- Communities beyond eight share hues (deterministically); the legend teaches
  the encoding rather than indexing every community — the Modules sidebar is
  the index.
- Procedural planet surfaces ("almost real planets") remain a Phase-3
  decision under the "no elaborate game art before the loop teaches well"
  Non-Goal; this release ships hued bodies and class rings only.
- The Phase 1 milestone still awaits human tester evidence (issue #13); this
  release does not claim it.
